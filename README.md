<summary><b>AtomSupport — Real-Time Video Support Platform</b></summary>

Live peer-to-peer video support platform where agents annotate customer screens in real time and generate AI session summaries with one click.

| Aspect | Details |
|--------|---------|
| **Stack** | Next.js 14, TypeScript, Tailwind CSS, WebRTC, Supabase, Gemini API |
| **Scale** | Supports 100+ concurrent sessions, WebSocket realtime sync |
| **Performance** | P2P video (zero-latency annotation), <100ms chat latency |
| **Security** | Supabase Auth (agent login), row-level security, encrypted file storage |
| **Impact** | Enables agents to resolve issues 3x faster with visual guidance |
| **Repository** | [GitHub](https://github.com/vikramchahall/atom-support) |

**Architecture**: Agent browser and customer browser communicate directly via WebRTC. Agent annotations rendered on HTML Canvas overlay. **Key Features**:
- Live camera feed with agent annotations (pen, arrow, rectangle, circle, laser)
- Numbered stamps for step-by-step guidance
- File sharing (images, PDFs, docs) with Supabase Storage
- AI-powered session summaries via Gemini 1.5 Flash
- Customer join via invite link (no account needed)
- Real-time chat with Supabase Realtime

<details>
<summary><b>AI/ML-Based Adaptive Traffic Signal Control — Sensor Fusion System</b></summary>

Adaptive traffic signal control using physical sensor fusion (IR + Ultrasonic) and XGBoost regression for real-time traffic analysis.

| Aspect | Details |
|--------|---------|
| **Stack** | Python, Flask, XGBoost, Sensor Fusion, Raspberry Pi 5, PostgreSQL |
| **Scale** | Real-time inference on edge device, processes 100+ sensor events/sec |
| **Performance** | Sub-100ms signal duration prediction, 85%+ accuracy on vehicle count |
| **ML Pipeline** | Feature engineering from raw sensor data, time-series feature extraction |
| **Impact** | Aligns with IRC standards; potential 15-20% reduction in average wait time |
| **Repository** | [GitHub](https://github.com/vikramchahall/traffic-signal-control) |

**Architecture**: Dual-gantry sensor fusion setup (IR + Ultrasonic). Raw sensor streams fed into XGBoost regression model. Model predicts optimal green-signal duration based on traffic composition, rush-hour status, road conditions. Raspberry Pi 5 runs inference. Flask REST API logs predictions to PostgreSQL. Real-time dashboard visualizes live traffic metrics.

**Key Features**:
- Multi-sensor fusion (IR for speed/classification, Ultrasonic for count/length)
- Vehicle count, speed, length, and classification estimation
- XGBoost-based signal duration optimization
- Edge inference with sub-100ms latency
- Real-time dashboard for traffic visualization

</details>
